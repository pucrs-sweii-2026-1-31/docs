# Frontend

![Next.js](../../images/next-js.svg){ class="tech-logo" }
### Next.js
[https://nextjs.org/](https://nextjs.org/){ class="tech-link" }

- **Quando usar:** quer estrutura robusta, roteamento organizado e possibilidade de SSR/SSG.
- **Pontos fortes:** App Router moderno, integração fácil com APIs, ótima organização de projeto, excelente para aplicações maiores.
- **Pontos de atenção:** maior complexidade inicial (rotas, estratégias de renderização, configuração).

---

![Angular](../../images/angular.gif){ class="tech-logo" }
### Angular
[https://angular.io/](https://angular.io/){ class="tech-link" }

- **Quando usar:** quer framework completo e altamente estruturado.
- **Pontos fortes:** arquitetura bem definida, DI nativo, RxJS poderoso, CLI madura, ideal para aplicações grandes e organizadas.
- **Pontos de atenção:** curva de aprendizado maior (TypeScript avançado, RxJS, decorators, módulos).

---

![Vite](../../images/vite.jpeg){ class="tech-logo" }
### Vite
[https://vitejs.dev/](https://vitejs.dev/){ class="tech-link" }

- **Quando usar:** quer ambiente moderno, leve e extremamente rápido para desenvolvimento de SPA.
- **Pontos fortes:** build rápido, hot reload muito eficiente, configuração simples.
- **Pontos de atenção:** não é um framework, exige escolher e organizar manualmente a arquitetura da aplicação.
