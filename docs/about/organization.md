# Organização

Os projetos deverão ser desenvolvidos em equipes de no mínimo 3 e no máximo 4 componentes.

Cada equipe será responsável por:

- Planejar a arquitetura da solução
- Manter seu backlog atualizado no board do Moodle
- Implementar os serviços
- Manter sua codebase atualizada no GitHub
- Documentar as APIs
- Garantir que os componentes funcionem de forma integrada
